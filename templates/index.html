CSS Unit Game

Absolute Units (Fixed Size)

  • px - Pixels (OG unit, always the same size)
  • pt - Points (print design vibes)
  • in - Inches (rarely used online)
  • cm - Centimeters (European flex)
  • mm - Millimeters (super precise flex)

Relative Units (Adaptive Kings)

  • % - Percentage (relative to parent - like inheriting your style)
  • em - Based on parent font size (family vibes)
  • rem - Based on root font size (consistent across site)
  • vw - Viewport width % (screen-width dependent)
  • vh - Viewport height % (screen-height dependent)
  • vmin - % of smaller viewport dimension (adaptive flex)
  • vmax - % of larger viewport dimension (bigger adaptive flex)

Elite Tip: Use rem for font sizes and spacing for consistent scaling. Use % and viewport units for layout. Pixels are only for borders and tiny details.

How To Not Be Basic

HTML Do's and Don'ts

  • Keep your HTML tags lowercase (uppercase is giving boomer energy)
  • Close all your tags (don't leave things hanging)
  • Use class names that actually make sense for what they do
  • Add alt text to images (accessibility = elite behavior)
  • Use semantic tags instead of div soup everywhere
  • Test your site on different browsers (Chrome isn't the only one)

CSS Glow Up Tips

  • Keep your CSS in separate files (organization = king behavior)
  • Group related styles together (not random scattering)
  • Use a CSS reset to avoid browser default chaos
  • Avoid using !important (it's like yelling - only do it when necessary)
  • Use shorthand properties to keep code clean
  • Build mobile-first (small screens first, then add for bigger screens)
  • Comment your complex CSS so future you doesn't get confused

Real Talk: The best websites aren't the ones with the most CSS tricks. They're the ones that load fast, work on any device, and make content easy to find. Focus on the user experience first, fancy animations second.
